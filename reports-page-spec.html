1 · Overview

The Reports page provides two capabilities:

  • Report Generator — select a scope (by owner or by environment list), a time period (daily, weekly, or custom date/time range), then trigger a Lambda that queries Neptune and generates an HTML health report stored in S3.
  • Report History — browse all past reports stored in S3, organised in a collapsible Year → Month → Date tree. Reports can be opened in the browser (via pre-signed S3 URL) or downloaded as HTML files.
Reports are generated server-side by a dedicated Lambda (REPORT_API_URL). The output is an HTML file stored in S3. The frontend never processes report content — it only receives metadata and a pre-signed URL.

3 · API Endpoints

GET Generate Report

{CONFIG.REPORT_API_URL}/generate-report

Headers: x-api-key: {API_KEY_REPORT}

Query params — Owner mode:

owner=platform-team&reportType=weekly

Query params — Environments mode:

environmentIds=env-001&environmentIds=env-002&reportType=daily

Query params — Custom range:

owner=platform-team&reportType=daily
&startDate=2025-06-01T00%3A00%3A00.000Z&endDate=2025-06-07T23%3A59%3A59.000Z

environmentIds is a repeated query parameter (multi-value). Lambda reads it from event.multiValueQueryStringParameters.environmentIds.

Success response 200: 

{
  "success": true,
  "presignedUrl": "https://s3.amazonaws.com/…?X-Amz-Signature=…",
  "owner": "platform-team",
  "reportType": "weekly",
  "environmentCount": 12,
  "alertCount": 3,
  "durationMs": 4820
}

Error response:

{
  "success": false,
  "message": "No environments found for owner: xyz"
}

GET List Owners

{CONFIG.REPORT_API_URL}/list-owners

Headers: x-api-key: {API_KEY_REPORT}

Success response 200: 

{
  "owners": ["platform-team", "data-team", "infra-team"]
}

GET Get Report (Pre-signed URL)

{CONFIG.REPORT_API_URL}/get-report?reportKey=reports%2Fplatform-team%2Fweekly%2F2025-06-15.html

Headers: x-api-key: {API_KEY_REPORT}

Success response 200: 

{
  "reportKey": "reports/platform-team/weekly/2025-06-15.html",
  "presignedUrl": "https://s3.amazonaws.com/…?X-Amz-Signature=…"
}

GET List Reports (History)

{CONFIG.REPORT_API_URL}/list-reports (optional: ?owner=platform-team)

Headers: x-api-key: {API_KEY_REPORT}

Success response 200: 

{
  "reports": [
    {
      "key": "reports/platform-team/weekly/2025-06-15_report.html",
      "lastModified": "2025-06-15T08:30:00.000Z",
      "size": 48320
    }
  ]
}

Reports are sorted client-side by lastModified descending. The key is parsed to extract owner, type, and filename (path segments: reports/{owner}/{type}/{filename}).

4 · Report Generator Card

4.1 Filter Mode Toggle

  • Two radio buttons: Owner (default) and Environments.
  • When "Owner" is selected: show the owner single-select MS dropdown (id="rpt-owner-ms", data-single="true"), hide the environments multi-select.
  • When "Environments" is selected: hide the owner dropdown, show the environments multi-select (id="rpt-env-ms") with "Select All" / "Clear All" actions.
  • The filter mode toggle calls toggleReportFilterMode().

4.2 Owner Dropdown

  • Single-select MS-Dropdown (data-single="true").
  • Populated at tab init from the owners known in the system (load from the list-reports endpoint or a dedicated owners list).
  • Required when filter mode is "Owner" and Generate is clicked.

4.3 Environments Multi-Select

  • Multi-select MS-Dropdown (standard, no data-single).
  • Includes "Select All" and "Clear All" links in the dropdown panel.
  • At least one environment must be selected when mode is "Environments" and Generate is clicked.

4.4 Report Type

Value Label Behaviour
daily Daily Lambda selects the last 24 h of data
weekly Weekly (default) Lambda selects the last 7 days
custom Custom Range Shows the Custom Range datetime inputs; both sent as ISO-8601 UTC strings

Changing report type to "custom" calls toggleCustomRange() which reveals the custom-range box.

4.5 Custom Range Validation

  • Both start and end datetime inputs must be filled.
  • new Date(startDate) < new Date(endDate) must be true. If not: show alert "Start date must be before end date".
  • When custom type is selected: the API body sends reportType: "daily" + startDate + endDate (custom is a front-end concept only).

4.6 Generate Button

  • ID: #rpt-generate-btn. Label text in #rpt-btn-text, spinner in #rpt-btn-spinner.
  • On click: validate filter mode, validate date range if custom, then GET with query params.
  • During generation: disable button, show spinner, set btn text to "Generating…", set status text "Connecting to Neptune and generating report…".
  • Always re-enable the button in a finally block.

5 · Generated Report Result Card

ID: #rpt-result. Hidden by default; displayed on successful generation. Shows:

Field Element ID Value Source
Owner #rpt-res-owner data.owner from API response
Type #rpt-res-type "Custom Range" if custom, else data.reportType
Environments #rpt-res-envs data.environmentCount
Alerts #rpt-res-alerts data.alertCount
Generated In #rpt-res-duration data.durationMs + "ms"

Two action buttons:

  • Download HTML — fetches the S3 pre-signed URL, receives the HTML blob, triggers a client-side download. File name: ems-report-{owner}-{type}-{YYYY-MM-DD}.html. No server call beyond the S3 fetch.
  • Open in Browser — calls window.open(currentReportUrl, '_blank'). The module-level variable currentReportUrl holds the S3 pre-signed URL from the last generate call.
S3 pre-signed URLs expire (typically 1 hour). Warn the user if they attempt to open/download a report that was generated in a previous session.

6 · Report History Card

6.1 Owner Filter

  • A <select id="rpt-history-owner"> with "All Owners" as the default option.
  • Additional owner options are populated from the distinct owners found in the report list.
  • On change: calls loadReportHistory() — triggers a fresh GET with the new owner query parameter.

6.2 Refresh Button

Calls loadReportHistory(). Re-fetches the full report list from S3 and rebuilds the tree.

6.3 Table Columns (within each date group)

Column CSS Class Width Data Source

7 · History Tree Structure

Reports are grouped in a three-level collapsible hierarchy. All levels are collapsed by default. Clicking a header toggles its children.

7.1 Tree Levels

Level CSS Class Toggle Label Format
Year .rh-year toggleRhGroup(yrId) 2025 (N reports)
Month .rh-month toggleRhGroup(mId) June 2025 (N)
Date .rh-date toggleRhGroup(dId) WED, 18 JUN 2025 (N)

7.2 Pagination

  • Paginates on date groups (5 date groups per page, constant RPT_PAGE_SIZE = 5).
  • Module-level variables: _rptHistoryData (full list), _rptHistoryPage (0-indexed current page).
  • Navigation buttons: ← Prev · "Page N / Total" · Next →.
  • Prev disabled on page 0; Next disabled on last page.

7.3 Date Timezone

All date grouping and display uses the timezone configured in CONFIG.TIMEZONE (default: MST). Use toLocaleDateString("en-CA", { timeZone: CONFIG.TIMEZONE }) to derive the YYYY-MM-DD date key.

7.4 Collapse/Expand Animation

  • Collapsed body: .rh-body.collapsed { display: none }.
  • Arrow indicator: .rh-arrow rotates 90° when open (.rh-arrow.open { transform: rotate(90deg) }).
  • Toggle function swaps the collapsed class and the open class on the arrow.

8 · Client-Side State

Variable Type Purpose
currentReportUrl string Pre-signed S3 URL from the most recent successful generate call. Used by Download and Open buttons.
_rptHistoryData Array Full flat array of report objects loaded from list-reports. Source for pagination and tree rendering.
_rptHistoryPage number Current 0-indexed page in the history tree pagination.
Filter mode radio "owner" | "environments" Controls which input section is visible and which field is validated/sent.

Report generation can take 30–120 seconds depending on environment count and alert volume. Show a spinner and a status message. Do not use a fixed timeout — wait for the response.

10 · Error Handling

Scenario UI Response
Owner not selected (owner mode) alert("Please select an owner") — halt generate.
No environments selected (env mode) alert("Please select at least one environment") — halt generate.
Custom range: missing start or end alert("Please select both start and end date/time") — halt.
Custom range: start ≥ end alert("Start date must be before end date") — halt.
Generate API returns success: false Show error text in #rpt-status: data.message in red. Hide result card.
Generate network error Show error text in #rpt-status: "Error: {e.message}" in red.
History load failure Show error text in #rpt-history-list: "Error loading history: {e.message}" in red.
HTML download failure alert("Download failed: " + e.message).
After a successful generate, automatically call loadReportHistory() so the new report appears at the top of the history tree without requiring a manual Refresh click.
